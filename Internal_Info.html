🧱 Recommended Technical Modules / Files

1. record_block.py → Low-level storage layer

Purpose: Simulate the disk and block system.

Contents:

  • class Record: represents one employee entry
    • Attributes: name, ssn, department, address, phone, birthdate, sex, jobcode, salary, deleted_flag
    • Method: to_bytes() and from_bytes() (simulate fixed-length serialization)
  • class Block: represents a 512-byte block
    • Holds multiple Record objects until full
    • Methods:
      • add_record(record)
      • mark_deleted(ssn)
      • is_full()
      • display()

Used by: higher modules to access stored records.

Role: Foundation of the “file system”.

2. bptree_nodes.py → Core data structure layer

Purpose: Define the building blocks of the B+ tree.

Contents:

  • class Node: base class for tree nodes
    • Common attributes: keys, parent, is_leaf, etc.
  • class InternalNode(Node): handles pointers to child nodes
    • Methods: insert_key(), split()
  • class LeafNode(Node): stores data pointers (references to record/block)
    • Methods: insert_entry(), split(), borrow_from_sibling(), merge_with_sibling()

Used by: the BPlusTree controller.

Role: Defines node behaviors and relationships.

3. bplustree.py → Tree management layer

Purpose: Manage all B+ tree operations.

Contents:

  • class BPlusTree: high-level manager that uses the node classes
    • Attributes: root, order_internal, order_leaf
    • Methods:
      • insert(ssn, record_ref)
      • delete(ssn)
      • find(ssn)
      • split_node(node)
      • print_tree(level_order=True)
  • Responsible for:
    • Insertion logic (splitting internal/leaf nodes)
    • Deletion logic (borrowing/merging)
    • Maintaining the root changes

Used by: the operations layer (next module).

4. file_index.py → Integration / Index layer

Purpose: Connect the file blocks with the B+ tree index.

Contents:

  • Class FileIndexManager:
    • Loads records from CSV
    • Inserts them into file blocks
    • Builds B+ tree on the SSN field
    • Keeps mapping between SSN ↔ (block_id, record_offset)
    • Handles updates when records are deleted

Methods:

  • load_csv(file_path)
  • initialize_tree()
  • insert_record(record)
  • delete_record(ssn)
  • show_blocks()
  • show_tree()

Used by: the main application interface.

Role: Links physical storage to logical indexing.

5. main.py → Application / Control layer

Purpose: Coordinate all modules for the full workflow.

Contents:

  • Command-line or sequential execution of:
    1. Load and store records.
    1. Build initial B+ tree.
    1. Insert new records (#27, #14, #22).
    1. Delete records (#11, #6, #3).
    1. Print final structures.

Optional additions:

  • Command menu for manual testing.
  • Export tree visualization using Graphviz or text-based ASCII.
